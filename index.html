Búsqueda Semántica

¿Qué es la búsqueda semántica? A diferencia de la búsqueda por palabras clave, la búsqueda semántica convierte tu consulta en un vector de 384 dimensiones mediante el modelo all-MiniLM-L6-v2. Luego calcula la similitud coseno entre ese vector y los chunks almacenados en MongoDB Atlas. Un score de 1.0 indica identidad perfecta, 0.7+ indica alta relevancia semántica.

Evaluaciones RAGAS BONUS

Métricas RAGAS:
  • Faithfulness: ¿La respuesta es fiel al contexto recuperado? (0–1)
  • Answer Relevancy: ¿La respuesta es relevante para la pregunta? (0–1)
  • Context Recall: ¿El contexto recuperado cubre la respuesta correcta? (0–1)
Usa Groq (Llama 3.1) como LLM juez para las métricas RAGAS. Las filas en verde indican evaluaciones RAGAS reales.
Procesa 22 preguntas × 3 estrategias — puede tardar 5–10 min

Implementación RAGAS Completa

# Ejecutar desde: cd python && python evaluacion_ragas.py
# O usar el botón de arriba para ejecutar desde la API

from ragas import evaluate, EvaluationDataset
from ragas.metrics import Faithfulness, AnswerRelevancy, ContextRecall
from ragas.llms import LangchainLLMWrapper
from ragas.embeddings import LangchainEmbeddingsWrapper
from langchain_groq import ChatGroq

# Configurar Groq como LLM juez
groq_llm = LangchainLLMWrapper(ChatGroq(model="llama-3.1-8b-instant"))

metrics = [
    Faithfulness(llm=groq_llm),       # ¿respuesta fiel al contexto?
    AnswerRelevancy(llm=groq_llm),    # ¿respuesta relevante a la pregunta?
    ContextRecall(llm=groq_llm),      # ¿contexto cubre la respuesta correcta?
]

# Dataset con columnas: user_input, response, retrieved_contexts, reference
dataset = EvaluationDataset.from_list([
    {
        "user_input": "¿Se permiten mascotas?",
        "response": respuesta_llm,
        "retrieved_contexts": lista_de_chunks,
        "reference": respuesta_ground_truth,
    }
])

result = evaluate(dataset=dataset, metrics=metrics)
print(result.to_pandas())
